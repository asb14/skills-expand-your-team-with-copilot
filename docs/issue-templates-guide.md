# Issue Templates Guide for Teachers

## Overview

The Mergington High School Management System now has structured issue templates to help teachers request changes and get support. These templates ensure that all necessary information is provided so that issues can be resolved quickly and accurately.

## Available Templates

### 🎯 Add New Activity
**Use when:** You want to add a new extracurricular activity to the system

**What you'll need to provide:**
- Activity name and description
- Meeting days and times
- Maximum number of participants
- Teacher sponsor information
- Any special requirements

**Example:** Adding a "Photography Club" that meets Wednesdays 3:15-5:00 PM

---

### ✏️ Modify Existing Activity  
**Use when:** You need to change details about an existing activity

**What you can modify:**
- Meeting schedule (days/times)
- Activity description
- Maximum participants
- Teacher sponsor information

**Important:** Student registrations will be preserved during modifications

**Example:** Changing Chess Club meeting time from 3:15 PM to 4:00 PM

---

### 🐛 Bug Report
**Use when:** Something isn't working correctly in the system

**What to include:**
- Which part of the system has the problem
- What happened vs. what should have happened
- Step-by-step instructions to reproduce the issue
- Any error messages you see

**Example:** Students can't register for Drama Club - getting error message

---

### 💡 Feature Request
**Use when:** You want new functionality added to the system

**What to describe:**
- What new feature you need
- What problem it would solve
- Who would use it
- How it should work

**Example:** Adding a calendar view to see activity schedules by date

---

### 🆘 Technical Support
**Use when:** You need help using the system or understanding how something works

**What to include:**
- What you're trying to accomplish
- What you've already tried
- How urgent your need is

**Example:** "How do I unregister a student who transferred schools?"

## How to Create an Issue

1. Go to the repository on GitHub
2. Click the "Issues" tab
3. Click "New Issue"
4. Choose the appropriate template
5. Fill out all required fields
6. Click "Submit new issue"

## Tips for Getting Quick Help

✅ **Do:**
- Use the most specific template for your request
- Fill out all required fields completely
- Provide clear, detailed descriptions
- Include specific examples when possible

❌ **Don't:**
- Leave required fields empty
- Use vague descriptions like "it doesn't work"
- Mix multiple unrelated requests in one issue
- Skip the templates and create blank issues

## Getting Started

**New to the system?** Start with the Technical Support template to ask questions about how things work.

**Need something changed?** Use the specific templates for adding or modifying activities.

**Something broken?** Use the Bug Report template with detailed steps to reproduce the problem.

**Have ideas for improvements?** Use the Feature Request template to describe what you'd like to see added.

## Support

If you're not sure which template to use, start with the Technical Support template and describe what you're trying to accomplish. The support team can help guide you to the right solution.
